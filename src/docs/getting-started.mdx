# Getting Started

## Choose your setup

Bloum ships its CSS as three separate stylesheets. Which ones you load depends on
whether your project already uses Tailwind CSS.

| Stylesheet          | Standalone | With Tailwind | Contains                                     |
| :------------------ | :--------- | :------------ | :------------------------------------------- |
| `reset.min.css`     | ✅ Yes     | ❌ No         | Cross-browser normalization                  |
| `bloum.min.css`     | ✅ Yes     | ✅ Yes        | Design tokens and all component styles       |
| `utilities.min.css` | ✅ Yes     | ❌ No         | Layout helpers (flex, grid, spacing, sizing) |

**Standalone** — Bloum has no dependency on Tailwind. The reset gives you the
same normalization a utility framework's preflight would, and the utilities give
you just enough layout classes to arrange components without pulling in a whole
framework.

**With Tailwind** — load only `bloum.min.css`. Tailwind's preflight already
covers the reset, and Tailwind's own utilities already cover the helpers, so the
other two files are redundant. See
[Tailwind CSS](/?path=/docs/documentation-tailwind-css--docs) for details.

## Installation

### CDN

The fastest way to get started. Add the following to your HTML `<head>` tag:

```html
<!-- Standalone (no Tailwind) -->
<link href="https://unpkg.com/bloum/dist/reset.min.css" rel="stylesheet" />
<link href="https://unpkg.com/bloum/dist/bloum.min.css" rel="stylesheet" />
<link href="https://unpkg.com/bloum/dist/utilities.min.css" rel="stylesheet" />

<script src="https://unpkg.com/bloum/dist/bloum.bundle.min.js" defer></script>
```

If you already use Tailwind, keep only the middle stylesheet:

```html
<!-- With Tailwind -->
<link href="https://unpkg.com/bloum/dist/bloum.min.css" rel="stylesheet" />

<script src="https://unpkg.com/bloum/dist/bloum.bundle.min.js" defer></script>
```

`bloum.bundle.min.js` is a self-contained browser build that exposes a global
`Bloum` object. Call `init()` on it once the DOM is ready to register all
components:

```html
<script>
  document.addEventListener("DOMContentLoaded", () => Bloum.init());
</script>
```

> **Note:** For production, pin a specific version to avoid unexpected breaking
> changes — replace `bloum` with `bloum@0.0.1` in each URL above _(using the
> latest version number)_.

### Package Manager

For projects using a build tool, install Bloum with your favorite package
manager:

```bash
# NPM
npm install bloum

# Yarn
yarn add bloum

# Pnpm
pnpm add bloum

# Bun
bun add bloum
```

## Usage

### 1. Import Styles

In your application's entry point, import the stylesheets for your setup.

**Standalone (no Tailwind)** — import all three:

```js
import "bloum/dist/reset.min.css";
import "bloum/dist/bloum.min.css";
import "bloum/dist/utilities.min.css";
```

**With Tailwind** — import only Bloum's core stylesheet, after Tailwind:

```js
import "tailwindcss";
import "bloum/dist/bloum.min.css";
```

Bloum's styles are organized with native CSS layers
(`@layer theme, base, components, utilities`), so the cascade stays predictable
regardless of import order. The order above is the one Bloum's own Storybook
environment uses, and the one we develop against.

### 2. Initialize Bloum

Import the `init` function and call it to register all components globally:

```js
import { init } from "bloum";

init();
```

### 3. Use Components

Now you can use the components in your HTML. Here is a simple example to verify
your installation:

```html
<button class="btn btn-primary">Hello Bloum!</button>
```

If you see a styled button, congratulations! You are ready to build.

## Upgrading from an earlier version

Earlier releases bundled a reset and a handful of helper classes directly into
`bloum.css`. Both are now separate, opt-in stylesheets. If you are upgrading and
are **not** using Tailwind, add the two new imports:

```js
import "bloum/dist/reset.min.css"; // new
import "bloum/dist/bloum.min.css";
import "bloum/dist/utilities.min.css"; // new
```

In particular, `sr-only`, `text-light`, `hstack` and `vstack` moved out of
`bloum.css` and now live in `utilities.min.css`.
